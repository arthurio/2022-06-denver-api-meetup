How and why you should leverage low code solutions for your notification system

I'm Arthur Rio, CTO at Notivize. We are a low code notification system platform.

This is a mouthful of a title, what's important here?

  • Notification systems are pretty important, it can be powerful tool or a disaster.
  • Low code. Doesn't mean no code. We try to make it as easy as possible for project managers and marketing to do their part without having to constently ping the dev team.

Start with a few options for each major channels. We let you start with few free sms/emails but you'll quickly need to plug in your own integration. We are looking into provisioning.

Rules engine

We focus on 2 things:

  • Conditional: Send only if condition is met
  • Schedule: event reminder, survey, delay
  • Frequency: Only once (Welcome email), every time (OTP), every time threshold is met (temperature alert)
For every time, Stop after X notifications, etc.

Automations

Group multiple notifications under one snippet to setup cadences, like a user onboarding for the first 7 days.
