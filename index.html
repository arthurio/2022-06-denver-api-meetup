# Why did we start Notivize? ![Notivize](images/notivize.png)
## Everybody needs notifications - Onboarding new users - Internal messages - Reminders - One time passwords - Status updates - etc. Note: How many of you have had to code notifications for your app? What did that entail? Custom logic? Building APIs? Maintaining a database? Templates? Crons?
## You quickly need a solution that can scale Note: For any startup, it's probably fine to start with hardcoded one off notifications, but what happens when you need to scale that to more channels, vendors, track analytics, customize text, ensure that you don't send twice the same notification, etc.?
## Multiple channels and vendors Note: Like Monk would say, "it's a gift... and a curse!" It's awesome to have so many APIs, but... Every API is different! What if you could just configure your new provider via a wizard and flip the switch?
# What key features have we built?
## Integrations - **Sms**: Twilio, AWS SNS, Ringcentral - **Email**: Sendgrid, AWS SES - **Platforms**: Slack, Discord - **Mobile** push: Firebase (Google), Apns (Apple)
## Rules engine - **Conditional**: Send only if condition is met - **Frequency**: Send every time, only once, every time threshold is met, stop after X notifications, etc.
## "Time travel" - Scheduled event reminder - Survey - Delay - Time of the day Note: If you have ever had to work with dates, time, and timezones, you must know how challenging it can become.
## Analytics - Sent - Failed - Excluded - Delivered - Scheduled Note: Users can also go look at the provider's data for more detailed analytics
## Message customization - UI to edit templates + Jinja or Mustache - Language variations Note: This is somewhat decoupled from the snippet. A project manager or marketing person can go in and change the copy as long as they don't introduce new dynamic variables.
## Inspire - Collect what well known companies do and turn it into a library of templates - Case by case suggestions from our founders
# What are we focusing on right now?
## Automation Group multiple notifications under one snippet to setup cadences, like a user onboarding for the first 7 days.
## AI powered content - AI recommended messaging plans - AI generated copy for notifications
# What's next?
## Integrations - **More channels**: web push, other platforms - **More vendors**: Mailchimp, SendInBlue, Postmark, Telnyx, Campfire, etc. - Provision third party accounts
## Profiles - Enables more complex rules - Content creators can update messages without changing the snippet
## Deployment/setup - Terraform module - On site solution
## A/B testing Compare performances between 2 versions of the same notification, between different channels, etc.
## AI - Better integrated in the creation process - Smarter (it's at the toddler stage right now)
# Conlusion Don't build it yourself, but do use notifications to delight your users!
